# Architecture

The workflow is a Tdarr Local Flow Plugin chain. Plugins do not import each other directly; they communicate through `args.variables` and through CSV/JSON state files in Tdarr's config directory.

## High-level flow

```text
preflight / metadata
  → HDR detection
  → video sample extraction
  → CQ sweep encoding
  → VMAF/CAMBI scoring
  → CQ bracket retry when needed
  → best-parameter selection
  → final AV1 NVENC transcode
  → result export
  → CQ-learning update
```

## Main plugin roles

| Plugin | Role |
|---|---|
| `checkVideoCodec` | Skip or allow files based on source codec. |
| `checkFileLimits` | Enforce size/duration gates before expensive processing. |
| `detectGPUEncoder` | Detect available NVIDIA hardware encoders. |
| `fetchMediaMetadata` | Optional metadata enrichment from Plex/TMDB/TVDB, with filename fallback. |
| `checkHdrContent` | Detect HDR/PQ/HLG/Dolby Vision-related stream metadata. |
| `extractVideoSamples` | Extract representative samples and load learned CQ priors. |
| `testEncodingParameters` | Build CQ candidate ranges and encode test samples. |
| `calculateVMAF` | Score candidates with VMAF and CAMBI where supported. |
| `checkCQBracket` | Decide whether the tested CQ range brackets the target quality. |
| `checkCQRangeRetry` | Retry lower/higher CQ ranges when selection guards reject candidates. |
| `selectBestParameters` | Choose final CQ/preset using quality, size, CAMBI, and holdout checks. |
| `vmafOptimizedTranscode` | Run the final AV1 NVENC transcode. |
| `exportVMAFResults` | Append long-form scoring results. |
| `learnCQRange` | Append learning data and update CQ EMA/prior state. |
| `cleanupTempFiles` | Remove workdir artifacts. |
| `monitorTranscodeRetry` | Track final transcode retry behavior. |

## Runtime paths

Tdarr local plugins often exist in more than one runtime location. This repo stores source under:

```text
plugins/vmaf/<plugin>/1.0.0/index.js
```

A running Tdarr container usually needs copies under both:

```text
/app/server/Tdarr/Plugins/FlowPlugins/LocalFlowPlugins/vmaf/<plugin>/1.0.0/index.js
/app/Tdarr_Node/assets/app/plugins/FlowPlugins/LocalFlowPlugins/vmaf/<plugin>/1.0.0/index.js
```

Use `tools/install-local-plugins.sh` or copy carefully to both paths.

## Learning state

The workflow can use two kinds of learning data:

- local private state generated by your own Tdarr runs
- aggregate seed priors shipped in `data/seed/`

The public seed files are not raw training rows. They are broad bucket summaries designed to make cold starts less painful while avoiding media-library disclosure.

## Flow import caveat

The exported flow references `checkFileAge`, but that plugin is not included in this repository. See [Installation](installation.md#5-resolve-the-checkfileage-caveat) for options.
